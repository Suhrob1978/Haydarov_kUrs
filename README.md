# Курсовая работа №1 — Клиент-серверное мобильное приложение «Бэклог фильмов» (Кинополка)

Клиент-серверное Android-приложение для ведения личного бэклога фильмов.

## Состав проекта

```
01-movie-backlog/
├── MovieBacklogApp/   — Android-клиент (Kotlin, Jetpack Compose, Hilt, Retrofit)
├── server/            — Сервер (Kotlin, Spring Boot 3, Spring Data JPA, H2)
└── Отчет/             — Отчёт (.docx) и диаграммы
```

## Технологический стек

| Часть    | Технологии |
|----------|-----------|
| Клиент   | Kotlin, Jetpack Compose, Navigation Compose
| Сервер   | Kotlin, Spring Boot 3
| Протокол | REST (HTTP/JSON), авторизация по JWT |

## Адрес сервера (телефон / эмулятор)

В `MovieBacklogApp/local.properties`:

```
server.url=http://10.0.2.2:8080/
```

- **Эмулятор** на том же ПК, где сервер — оставьте `10.0.2.2`.
- **Реальный телефон** — укажите IP компьютера с сервером в Wi‑Fi, например `http://192.168.1.13:8080/`.
- После смены адреса пересоберите приложение (Rebuild).

Постеры картинок грузятся через сервер (`/api/movies/{id}/poster`), чтобы работали на всех устройствах.
