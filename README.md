# Курсовая работа №1 — Клиент-серверное мобильное приложение «Бэклог фильмов» (Кинополка)

Клиент-серверное Android-приложение для ведения личного бэклога фильмов:
каталог, поиск, списки «Хочу посмотреть / Смотрю / Посмотрел», оценки,
заметки и статистика. Хранение данных — на сервере.

## Состав проекта

```
01-movie-backlog/
├── MovieBacklogApp/   — Android-клиент (Kotlin, Jetpack Compose, Hilt, Retrofit)
├── server/            — Сервер (Kotlin, Spring Boot 3, Spring Data JPA, H2)
└── Отчет/             — Отчёт (.docx) и диаграммы
```

## Технологический стек

| Часть    | Технологии |
|----------|-----------|
| Клиент   | Kotlin, Jetpack Compose, Material 3, Navigation Compose, Hilt, Retrofit/OkHttp, Coil |
| Сервер   | Kotlin, Spring Boot 3, Spring Web, Spring Data JPA, Hibernate, H2 (файловая БД), JJWT, BCrypt |
| Протокол | REST (HTTP/JSON), авторизация по JWT |

## Требования

- **Android Studio** (актуальная версия, с JDK 21 в комплекте — JBR).
- **Android SDK** Platform 35 (compileSdk 35, minSdk 24).
- Эмулятор Android или физическое устройство.
- Доступ в интернет при первой сборке (загрузка зависимостей Gradle).

## Запуск — шаг за шагом

Приложение клиент-серверное, поэтому сначала запускается **сервер**, затем — **приложение**.

### 1. Запустить сервер

В терминале из папки `server`:

```bash
cd 01-movie-backlog/server
./gradlew bootRun
```

Сервер поднимется на `http://localhost:8080`. При первом запуске
автоматически создаётся файловая база H2 (папка `server/data/`) и
наполняется каталог фильмов. Консоль БД доступна на `http://localhost:8080/h2-console`
(JDBC URL: `jdbc:h2:file:./data/kinopolka`, пользователь `sa`, без пароля).

> Сервер можно также открыть как отдельный проект в IntelliJ IDEA / Android Studio
> и запустить класс `KinopolkaApplication`.

### 2. Запустить Android-приложение

1. В Android Studio: **File → Open** → выберите папку `01-movie-backlog/MovieBacklogApp`.
2. Дождитесь окончания Gradle Sync.
3. Выберите эмулятор и нажмите **Run ▶**.

Приложение обращается к серверу по адресу `http://10.0.2.2:8080` — это
`localhost` хост-машины с точки зрения **эмулятора** Android. Ничего менять не нужно.

> **Запуск на реальном устройстве:** замените в файле
> `MovieBacklogApp/app/src/main/java/com/kinopolka/app/di/AppModule.kt`
> константу `BASE_URL` на `http://<IP-вашего-компьютера>:8080/` (телефон и
> компьютер должны быть в одной сети Wi-Fi).

### 3. Пользоваться

Зарегистрируйте нового пользователя прямо в приложении (почта, пароль, имя)
и войдите. Каталог фильмов уже наполнен — добавляйте фильмы в бэклог,
меняйте статусы, ставьте оценки и смотрите статистику в профиле.

## Возможные проблемы

- **«Нет соединения с сервером»** — убедитесь, что сервер запущен (шаг 1) и
  отвечает на `http://localhost:8080/api/movies`.
- **Gradle Sync долго идёт** — это первичная загрузка зависимостей, дождитесь окончания.
- **Сбросить данные сервера** — остановите сервер и удалите папку `server/data/`.
